# Demo Recording Script

Recommended length: 2 to 4 minutes

Live demo URL: `https://tunnelsight.applications.co.zw/`

The goal is to prove that the project has a working prototype, while clearly separating working features from planned AI features.

## Recording Checklist

1. Open the live prototype.
2. Show the loading state while the 3D environment initializes.
3. Navigate around the 3D tunnel scene smoothly.
4. Switch to marker mode and place a risk marker.
5. Switch to measure mode and measure between two visible points.
6. Switch to selection/box-select mode and show the UI mode changing away from pure navigation.
7. Use screenshot/export capture if available in the interface.
8. Show the generated or visible risk/report panel if available.
9. End with a short status slide or spoken statement explaining what is already working and what is planned next.

## Suggested Spoken Script

This is the current early prototype of GAMIS, also called TunnelSight in the demo environment. The system is being developed as a mining and tunnel intelligence platform for 3D inspection, risk marking and future AI-assisted hazard analysis.

In the current prototype, the user can load and inspect a 3D tunnel scene in the browser. The viewer supports smooth navigation, visible loading states, risk-marker placement, basic distance measurement, selection modes and screenshot capture. These features demonstrate the user workflow for field review and safety reporting.

The current risk markers are demonstration markers and user annotations. Automated hazard detection is not yet being claimed as validated. During the challenge, the next development step is to connect a controlled image or video capture workflow to a reconstruction pipeline, then test first-pass hazard classification for visible hazards such as cracks, loose rock, water accumulation or unsupported areas.

The intended MVP is a decision-support tool. A user uploads controlled tunnel imagery, the system reconstructs or displays the environment, highlights selected hazards, allows measurements and annotations, and generates a preliminary report for review by a qualified mining or geotechnical professional.

Save the recording link and include it in the proposal or submission portal. Also capture at least three screenshots:

| Screenshot | Purpose |

## Avoid These Claims

- Do not say the system already prevents mine collapse.
- Do not say the AI model is already validated unless there are real metrics.
- Do not describe demo markers as real detected hazards.
- Do not present the system as a replacement for a qualified mining engineer.
